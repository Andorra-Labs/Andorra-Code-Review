The input event listener triggers render() on every keystroke without debouncing. For large configurations, this causes unnecessary JSON serialization and DOM updates on each character typed, leading to potential UI lag. Consider using requestAnimationFrame, setTimeout debouncing (e.g., 100-200ms delay), or a state batching approach to improve performance.

Suggestion:
```suggestion
  // Use debounced input handler to improve performance
  const debouncedRender = (() => {
    let timer;
    return () => {
      clearTimeout(timer);
      timer = setTimeout(() => render(), 100);
    };
  })();

  ["name", "url", "api_key"].forEach((field) => {
    const el = card.querySelector(`[data-field="${field}"]`);
    el.addEventListener("input", () => {
      const oldName = p.name;
      p[field] = el.value;
      if (field === "name") {
        const newName = el.value;
        for (const s of state.scanners) if (s.provider === oldName) s.provider = newName;
        if (state.arbiter.provider === oldName) state.arbiter.provider = newName;
        refreshWorkerSelects();
      }
      debouncedRender();
    });
  });
```
